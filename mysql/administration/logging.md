# Logging이란?
* MySQL에서 발생하는 다양한 이벤트를 기록하는 기능
* 문제 분석, audit, recovery의 기반이 되는 데이터
* 특징
    * 과거 이벤트를 기반으로 분석 가능
    * 장애 원인 추적 가능
    * 사용자 행위 감사(audit) 가능
* 한계
    * 실시간 상태는 알 수 없음
    * 로그 양이 많아질수록 성능에 영향을 미칠 수 있음

# Log 종류
## Error Log
* 특징
    * MySQL 서버의 상태 및 오류를 기록하는 로그
    * 서버의 시작, 실행, 종료되는 과정에서 발생한 문제를 기록
    * 가장 기본적인 운영 로그
    * 장애 분석 시 1순위로 확인하는 로그
* 기록 내용
    * 서버 시작/종료 메시지
    * 비정상 종료 관련 정보
    * 설정 오류
    * InnoDB 관련 오류/경고
    * replication 오류
    * 기타 warning, information 메시지
* 확인하는 경우
    * MySQL이 안 켜질 때
    * replication이 깨졌을 때
    * crash가 발생했을 때
    * 설정 파일 수정 후 정상 동작하지 않을 때
```sql
show variables like 'log_error';
```
```bash
tail -f /var/log/mysql/error.log
```
## General Log
* 특징
    * 클라이언트가 서버에 보낸 요청을 거의 그대로 추적하는 로그
    * mysqld가 무엇을 하고 있는지에 대한 일반 기록
    * 클라이언트 연결/해제와 클라이언트가 보낸 각 SQL문을 기록
    * 누가 어떤 쿼리를 보냈는지 확인할 수 있음
    * 성능 분석용 로그가 아님
    * 로그 양이 많아질수록 성능 부담이 큼
        * 운영 상에서는 일반적으로 상시 활성화는 지양함
    * 실행 결과가 아닌 요청 받은 SQL문을 기록
    * select도 기록됨
* 기록 내용
    * 클라이언트-서버 연결/해제 메시지
    * 클라이언트가 보낸 SQL문
    * 어떤 쿼리를 보냈는지에 대한 추적
* 확인하는 경우
    * 애플리케이션이 실제로 어떤 SQL을 보내는지 확인할 때
    * ORM이 예상과 다른 쿼리를 날리는지 확인할 떄
    * 특정 클라이언트 동작을 디버깅할 때
## Slow Query Log
* 특징
    * 느린 쿼리를 찾기 위한 성능 분석 로그
    * 성능 튜닝에 가장 실용적
    * 옵션을 통해 기록할 쿼리를 제한할 수 있음
        * `long_query_time`
            * 쿼리 실행 시간 기준
            * 설정값보다 오래 걸리는 쿼리를 기록
            * 너무 낮게 잡으면 로그 양 폭발
            * 너무 높게 잡으면 slow query 추적 불가
            * 보통, 개발 시에는 `0.1 ~ 0.5`, 운영 상에서는 `1 ~ 2`초로 설정
        * `min_examined_row_limit`
            * 쿼리가 검사한 row 수 기준
            * 설정값 이상의 row를 검사한 쿼리를 기록
            * 비효율적으로 row를 많이 읽는 쿼리를 걸러내는 필터
            * 성능 튜닝 대상 선별에 매우 중요
            * full scan 같은 쿼리를 잡을 수 있음
        * `log_queries_not_using_indexes`
            * 인덱스 미사용 쿼리를 기록하는 옵션
            * 실행 시간, row 검사 조건과 별개로 인덱스를 사용하지 않으면 무조건 기록
            * 느린 쿼리보다는 나쁜 쿼리를 기록하는 로그 옵션
            * 작은 테이블이나 인덱스가 필요 없는 쿼리도 기록되기 때문에 로그 양이 증가할 수 있음
                * 보통 운영 시에 제한적으로 사용
                * 테스트/튜닝 환경에서 주로 사용
* 기록 내용
    * 오래 걸린 쿼리
    * 실행 시간
    * 검사한 row 수
    * 옵션에 따라 인덱스 미사용 쿼리 등
* 확인하는 경우
    * 튜닝 후보를 찾을 때
    * explain 분석 대상을 고를 때
    * 응답이 느린 서비스 원인을 찾을 때
